# Magic Vibes - Склад

Веб-панель для личного склада: цены из PriceMaster, сверка с Ozon и Яндекс Маркетом и ручная отправка обновлений только для выбранных позиций.

## Что умеет сейчас

- берёт активные товары из `OfferRows`;
- считает, что цена PriceMaster указана в долларах;
- считает рублёвую цену по формуле `USD × курс × коэффициент`;
- курс USD/RUB берёт автоматически и кеширует на 6 часов;
- по умолчанию использует отдельные коэффициенты: Ozon `1.70`, Яндекс `1.60`;
- для каждого товара склада можно задать свою наценку (иначе берётся коэффициент по маркетплейсу из `.env`);
- сверяет товары по артикулу: `OfferRows.NativeID = offer_id`;
- на вкладке «Личный склад» показывает расчёт и текущие цены маркетплейсов;
- ничего не отправляет автоматически;
- отправляет цены только для выбранных товаров склада и только после подтверждения;
- закрывает сайт логином и паролем;
- показывает текущую цену маркетплейса, если API её вернул;
- для Ozon цена в интерфейсе совпадает с кабинетом: берётся из `POST /v5/product/info/prices` (поле цены и при необходимости акции селлера); при первом открытии склада в сессии браузера выполняется фоновое обновление цен с маркетплейсов.

## Быстрый запуск

```powershell
npm.cmd install
npm.cmd start
```

После запуска откройте:

```text
http://localhost:3000
```

Проверка, что процесс жив (`200 OK`): `GET http://localhost:3000/health`

В **production** задайте `NODE_ENV=production` и надёжный `APP_SESSION_SECRET` (иначе сервер не стартует). На вход действует ограничение числа попыток с одного IP.

Логин и пароль задаются в `.env`:

```text
APP_USER=admin
APP_PASSWORD=...
```

## Передача цен

Вкладка «Личный склад»:

1. При необходимости проверьте курс USD/RUB (обновляется автоматически раз в 6 часов; при ручном изменении учитывается введённое значение).
2. Убедитесь, что коэффициенты наценки для Ozon и Яндекс заданы в настройках (или используются значения по умолчанию из `.env`).
3. Отфильтруйте товары по маркетплейсу, поиску и статусу Ozon при необходимости.
4. Отметьте галочками товары, для которых нужно обновить цену на маркетплейсе (кнопка `Выбрать изменения` подсвечивает позиции, где расчётная цена отличается от текущей).
5. Нажмите `Обновить цены`.
6. Подтвердите отправку во всплывающем окне.

Без выбора товаров и подтверждения сайт не отправляет цены.

Формула:

```text
цена PriceMaster в USD × курс USD/RUB × коэффициент наценки
```

Поле наценки в карточке товара на складе переопределяет значение по умолчанию для этой позиции. Для нескольких выбранных товаров можно задать наценку разом через форму «Наценка для выбранных».

Для Ozon используется:

```text
POST /v1/product/import/prices
```

Для Яндекс Маркета используется:

```text
POST /v2/businesses/{businessId}/offer-prices/updates
```

## Как считается изменение

Позиция прайса определяется ключом:

```text
поставщик + артикул + штрихкод + название
```

Это сделано потому, что в базе встречаются пустые или повторяющиеся артикулы. Если нужно считать строго по `NativeID`, правило можно изменить в функции `offerKey()` в `server.js`.

## Где хранятся данные сайта

Сайт не меняет базу Price Master. Он только читает ее.

Локальные файлы:

```text
data/snapshot.json   последний снимок для сравнения
data/history.jsonl   история всех найденных изменений
```

Папка `data/` добавлена в `.gitignore`, чтобы рабочая история не попадала в репозиторий.

```powershell
npm.cmd run inspect:db
```

## Запуск на сервере

Для постоянной работы лучше использовать `pm2`:

```powershell
npm.cmd install -g pm2
pm2 start server.js --name price-master-monitor
pm2 save
```

Для публикации наружу обязательно:

- поменять `APP_PASSWORD`;
- оставить MySQL-пользователю только права на чтение, если это возможно;
- поставить HTTPS через nginx/IIS/reverse proxy;
- в `PUBLIC_BASE_URL` указать внешний HTTPS-адрес.

### Продакшен: proxy, загрузки, логи, тесты, бэкапы

- **Trust proxy**: за nginx укажите `TRUST_PROXY_HOPS=1` (или фактическое число прокси), чтобы ограничение попыток входа считалось по IP клиента, а не по IP прокси.
- **Загрузки картинок**: `UPLOAD_MAX_FILES_PER_SESSION` (по умолчанию 200 за 24 ч на сессию), `UPLOAD_RETENTION_DAYS` (удаление файлов старше N дней), `UPLOAD_MAX_DISK_MB` (срез самых старых файлов при переполнении).
- **Логи**: в production строки лога — JSON в stdout; ротацию делайте через pm2, systemd или `logrotate` на перенаправленный файл. Явно: `LOG_JSON=true`.
- **Smoke-тесты**: `npm test` (приложение не открывает порт, используется экспорт `app`).
- **Бэкап каталога `data/`**: `powershell -File scripts/backup-data.ps1` — копия в `backups/<метка-времени>/` (папка в `.gitignore`).
- **Долгосрочно**: разнести `server.js` по модулям маршрутов; упростить дубли в `styles.css` (см. комментарий в начале файла).

## Личный склад и ежедневное обновление

В разделе `Личный склад` товары можно фильтровать отдельно по `Ozon` и `Yandex Market`, привязывать к ним несколько артикулов PriceMaster, указывать ключевое слово и поставщика. Система выбирает доступного поставщика с учетом цены, наличия, приоритета и стоп-листа.

Поставщики управляются во вкладке `Поставщики`. Если поставить поставщика на стоп, его найденные позиции в PriceMaster считаются недоступными и не участвуют в расчете новой цены. После включения поставщика обратно цены снова берутся из актуальных данных PriceMaster.

Ежедневное обновление включено настройками:

```text
DAILY_SYNC_ENABLED=true
DAILY_SYNC_TIME=11:00
```

По расписанию сервер делает снимок PriceMaster, обновляет личный склад из маркетплейсов и сохраняет статус в `data/daily-sync.json`. В интерфейсе есть кнопка `Запустить сейчас` для ручной проверки.

## Выгрузка товаров

Страница `/ozon-product.html` хранит все основные поля карточки Ozon: артикул, название, описание, категорию, цену, НДС, габариты, вес, штрихкоды, изображения и JSON-атрибуты. Кнопка `Сохранить в личный склад` сохраняет draft локально, а `Отправить товар в Ozon` отправляет карточку в Ozon Seller API после подтверждения.

Страница `/yandex-product.html` делает то же для Yandex Market: SKU, кабинет ЯМ, категория, производитель, описание, изображения, штрихкоды, цена и дополнительные JSON-поля.

В обеих формах есть импорт изображений. Файлы PNG, JPG, WEBP и GIF до 10 МБ сохраняются в `public/uploads/images`, после чего URL автоматически добавляется в поля изображений. Папка `/uploads/images/` открыта без авторизации, чтобы маркетплейсы могли скачать картинки по ссылке; на продакшене задайте лимиты и очистку через переменные окружения (см. выше). Для боевого сервера в `PUBLIC_BASE_URL` нужно указать внешний HTTPS-адрес сайта.

Из карточки товара в личном складе доступны отдельные действия:

- `Выгрузить в Ozon` через `POST /v2/product/import`;
- `Выгрузить в Яндекс` через `POST /v2/businesses/{businessId}/offer-mappings/update`.

Для Яндекс Маркета нужны `YANDEX_SHOPS_JSON`, категория ЯМ, описание, изображение и производитель. Если изображение ЯМ не указано, используются изображения из Ozon draft.

При синхронизации склад подтягивает существующие товары Ozon и Yandex Market. Для ссылок Ozon используется только публичный `productId`, `sku` или сохраненный `productUrl`; если публичного идентификатора нет, ссылка на поиск больше не показывается.

Страница `/product.html` поддерживает кнопку «Обновить цены» и параметр `?refreshPrices=1` (или `?refresh=1`) для принудительного запроса актуальных цен с маркетплейсов при открытии.
